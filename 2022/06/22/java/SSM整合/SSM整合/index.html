Spring层

1. 配置Spring整合MyBatis，我们这里数据源使用c3p0连接池；

2. 编写Spring整合Mybatis的相关的配置文件：spring-dao.xml

  • 关联数据库文件 database.properties
  • 数据库连接池设置
    • 连接池类型：
      • dbcp 半自动化操作 不能自动连接
      • c3p0 自动化操作（自动化的加载配置文件 并且可以自动设置到对象中）
      • druid
      • hikari springboot2.x默认
  • 配置SqlSessionFactory对象
  • 配置扫描Dao接口包，动态实现Dao接口注入到spring容器中

3. Spring整合service层：spring-service.xml

11. 常见报错

问题：bean不存在
步骤：

  1. 查看这个bean注入是否成功!

  2. Junit单元测试，看我们的代码是否能够查询出来结果!

  3. 问题，一定不在我们的底层，是spring出了问题!

  4. SpringMVC， 整合的时候没调用到我们的service层的bean;

    1. applicationContext. xml没有注入bean

    2. web. xml中，我们也绑定过配置文件! 发现问题，我们配置的是Spring -mvc. xml
      这里面确实没有service bean， 所以报空指针，应该改为:

    <param-value>classpath:applicationContext.xml</param-value>
